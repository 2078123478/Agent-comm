# Codex Task: 补一个超薄的 agent-comm 发送 API

你现在要在当前已存在的 CLI + helper 基础上，补一个**超薄**的发送 API。

## 目标
把当前已经存在的：
- `sendCommPing`
- `sendCommStartDiscovery`

提升成外部可直接调用的 HTTP API，但保持 **KISS**。

## 只做这两个 API
### 1. ping
建议形态：
- `POST /api/v1/agent-comm/send/ping`

请求体最小包含：
- `peerId`
- 可选：`senderPeerId`
- 可选：`echo`
- 可选：`note`

### 2. start_discovery
建议形态：
- `POST /api/v1/agent-comm/send/start-discovery`

请求体最小包含：
- `peerId`
- `strategyId`
- 可选 discovery 参数（按现有 helper 支持范围）
- 可选：`senderPeerId`

## 约束
1. **必须超薄**
   - 不要重造发送链路
   - 直接复用现有 helper / entrypoints

2. **不扩 scope**
   - 不要加更多命令
   - 不要加 ws/x402/paymaster/multi-chain
   - 不要顺手改协议

3. **保持现有鉴权风格**
   - API 应沿用当前 server 的认证方式

4. **同步文档**
   - README / 最小复用文档补充 API 用法
   - 但保持文档简洁

5. **补测试**
   - 至少验证这两个 API 能调用到正确发送入口
   - build/test 必须通过

## 输出
完成后请简短说明：
1. 新增了哪两个 API
2. 请求体最小格式是什么
3. build/test 结果
