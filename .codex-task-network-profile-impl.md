# Codex Task: 实施 network profile 第一阶段（T1）

你现在要开始实施 `NETWORK_PROFILE_TASK.md` 里的 T1：配置模型收敛。

## 先阅读
- `NETWORK_PROFILE_REQUIREMENT.md`
- `NETWORK_PROFILE_DESIGN.md`
- `NETWORK_PROFILE_TASK.md`

## 本轮目标
只做 T1，不要扩到后面太多：

### T1.1 定义 profile 数据结构
建议新增：
- `src/skills/alphaos/runtime/network-profile.ts`

至少定义：
- `xlayer-recommended`
- `evm-custom`
- defaults
- probes
- requiredUserInputs
- capabilityFlags

### T1.2 将 profile 映射到现有 config
重点修改：
- `src/skills/alphaos/runtime/config.ts`

要求：
- profile 能提供默认值层
- 现有显式 env 仍然优先
- `xlayer-recommended` 能收敛：
  - `196`
  - 推荐 rpc
  - `poll`
  - 推荐 auth mode
- `evm-custom` 保持自由输入

### T1.3 明确三类配置边界
在代码结构或注释/类型中明确：
- 可默认化
- 可自动探测
- 必须用户手填

## 约束
1. **不要扩 scope**
   - 不做 UI
   - 不做完整 probe pipeline
   - 不做 ws/x402/paymaster/AA 新实现

2. **保持向后兼容**
   - 现有 env/config 流程不要被破坏
   - 显式配置优先级高于 profile 默认值

3. **风格要求**
   - 保持 KISS
   - 不引入过重抽象
   - 让后续 T2/T3 易于接上
