# Codex Task: 用完整 code-simplifier 规则审视 agent-comm v0.1 当前实现

你现在要基于**不改变功能边界**的前提，用 code-simplifier 的规则审视当前 agent-comm v0.1 的实现。

## 完整 code-simplifier 规则（必须遵守）

1. **保持功能不变**
   - 只改写法，不改行为
   - 不扩 scope，不改 requirement / design 边界
   - 不引入 requirement/design 没要求的新能力

2. **减少嵌套**
   - 优先 early return
   - 避免深层 if-else
   - 如果一个函数里状态判断太多，优先拆 helper

3. **避免嵌套三元**
   - 用 switch 或清晰的 if/else 链替代
   - 不要为了紧凑牺牲可读性

4. **消除冗余**
   - 重复逻辑应提取为独立函数
   - 重复状态处理、重复错误构造、重复查询/映射逻辑都要重点关注

5. **清晰命名**
   - 变量/函数名应自解释
   - 避免“看源码才知道在干嘛”的命名

6. **明确优于紧凑**
   - 不追求一行代码
   - 优先让状态流转、失败边界、职责边界一眼能看懂

7. **不做大重构**
   - 重点找“值得做的小中型整理”
   - 不建议在当前阶段做高风险大搬家式重构

8. **优先关注运行时主链路**
   - runtime/bootstrap
   - config 启动校验
   - inbound/outbound 状态流转
   - 错误处理边界
   - 最小观测与 API 边界

## 你的目标
不是继续扩需求，也不是再做架构 review，而是检查：
- 现在的实现有没有明显可以更清晰、更易维护的地方
- 有没有嵌套过深、职责混杂、重复逻辑、命名不清、状态流转不直白的问题
- 有没有可以收敛但不改变行为的实现方案
