# Codex Task: 按 code-simplifier 规则实施安全的结构优化

你现在要基于前一轮 code-simplifier review，开始做**小中型、安全的结构优化**。

## 绝对约束
1. **不改变功能**
2. **不扩 scope**
3. **不改 requirement / design 边界**
4. **优先小步、安全、可验证的重构**
5. **每完成一批改动都要跑 build/test**

## 使用的 code-simplifier 规则
严格遵守以下规则：
- Preserve functionality
- Prefer clarity over cleverness
- Reduce nesting with early returns
- Avoid nested ternaries
- Eliminate duplicated logic where it improves readability
- Explicit > compact
- Small safe refactor > ambitious rewrite
- Focus on touched code and runtime critical path first
